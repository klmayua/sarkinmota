# Robots.txt Analysis for Sarkin Mota Autos

## Robots.txt Discovery & Access

An attempt was made to fetch the standard `robots.txt` file from the root of the domain:
- **URL**: `https://www.sarkinmota.ng/robots.txt`
- **Result**: `404 Not Found` (Status Code 404)

### Implications of 404 Status

1. **Default Crawler Behavior**: 
   In the absence of a `robots.txt` file, search engine web crawlers (like Googlebot, Bingbot, YandexBot) operate under the assumption that there are **no crawling restrictions** on the site. They are allowed to crawl and index any accessible URL they discover.
   
2. **Security & Privacy Risks**: 
   Since no paths are disallowed, search engines may index administrative, developmental, or user-private endpoints if they are exposed in front-end code (e.g. Next.js chunks) or if internal links accidentally point to them.
   
3. **SEO Performance**:
   - Web crawlers cannot locate references to a sitemap via the `robots.txt` file. They must rely on manual submissions to Google Search Console or discovery via external and internal links.
   - Crawl budget may be wasted on non-essential, dynamically generated, or duplicate pages (like sorting/filtering search parameters) since there are no rules to restrict them.

## Recommended Robots.txt Configuration

To improve SEO efficiency and secure the site, a standard `robots.txt` should be deployed:

```text
# Sarkin Mota Autos Robots.txt
User-agent: *
Allow: /
Disallow: /admin/
Disallow: /dashboard/
Disallow: /api/
Disallow: /_next/
Disallow: /tools/compare*

Sitemap: https://www.sarkinmota.ng/sitemap.xml
```

### Path Breakdown for Proposed Rules:
- **Allowed Paths**: All public pages, vehicle catalogs, news, about, contact, and tools.
- **Disallowed Paths**:
  - `/admin/`, `/dashboard/`: Protects administrative screens from appearing in search results.
  - `/api/`: Hides backend endpoints.
  - `/_next/`: Prevents indexing of Next.js framework internals.
  - `/tools/compare*`: Avoids indexing duplicate comparison pages generated by URL query strings.
- **Sitemap Reference**: Points crawlers directly to the sitemap XML for efficient URL discovery.
